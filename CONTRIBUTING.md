# Contributing to FossRadar

First off, thank you for considering contributing to FossRadar! It's people like you that make FossRadar such a great tool.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for FossRadar. Following these guidelines helps maintainers and the community understand your report, reproduce the behavior, and find related reports.

- **Use our [Bug Report Template](https://github.com/saurabhsharma2u/FossRadar/issues/new?template=bug_report.md)** to provide the necessary details.
- **Check if the bug has been reported already** in our issue tracker.
- **Provide detailed steps** to reproduce the issue. 
- **Include context** such as OS version, Node version, etc.

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When you are creating an enhancement suggestion, please:
- **Use our [Feature Request Template](https://github.com/saurabhsharma2u/FossRadar/issues/new?template=feature_request.md)** to provide the necessary details.
- Include a clear and descriptive title.
- Provide a step-by-step description of the suggested enhancement.

### Suggesting New Projects
If you've found an incredible FOSS alternative that isn't on the radar:
- **Use our [Suggest Project Template](https://github.com/saurabhsharma2u/FossRadar/issues/new?template=suggest_project.md)**.
- Provide the GitHub URL of the repository.
- Specify the appropriate category for the project.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`npm test`).
5. Include a description of the changes in the Pull Request.

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
